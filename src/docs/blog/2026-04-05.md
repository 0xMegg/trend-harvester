# 2026-04-05 — v4.0.0 업그레이드: BLOCK/WARN hook + 자동화 스크립트

> hook 시스템을 심각도 기반으로 분리하고, Task/Epic 자동화 스크립트를 추가하여 3-Role Workflow를 원커맨드로 실행 가능하게 만든 날.

---

## 배경

v3까지는 post-edit hook이 단순 PASS/FAIL 이진 판정이었다. 실전(DiveBase)에서 사용하다 보니, lint 경고처럼 즉시 수정해야 하는 것(BLOCK)과 리뷰어에게 맡겨도 되는 것(WARN)을 구분할 필요가 생겼다. 또한 매번 `/plan` → `/develop` → `/review`를 수동으로 호출하는 반복 작업을 자동화하고 싶었다.

---

## 작업 내역

### BLOCK/WARN hook 분리

`post-edit-check.sh`를 BLOCK(즉시 중단)과 WARN(경고만) 두 단계로 리팩토링. BLOCK은 `dart analyze` 에러, 금지된 패턴 사용 등 즉시 수정이 필요한 항목. WARN은 TODO 증가, 파일 크기 초과 등 리뷰어가 판단할 항목.

### auto-test runner hook 추가

`post-edit-test.sh` 신규 추가. 수정된 파일에 대응하는 테스트 파일이 있으면 자동으로 해당 테스트만 실행. 전체 테스트를 매번 돌리지 않아도 즉각적인 피드백 가능.

### run-task.sh / run-epic.sh 자동화 스크립트

`scripts/run-task.sh`: 단일 Task를 Plan → Develop → Review 순서로 자동 실행. APPROVE 시 커밋, REQUEST_CHANGES 시 중단.
`scripts/run-epic.sh`: Epic을 슬라이스로 분해한 뒤 각 슬라이스를 자동으로 3-Role 실행.
