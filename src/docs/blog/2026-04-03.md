## 회고

**한 줄 요약**: v3.0을 릴리스하고, 직접 써보면서 발견한 3가지 실전 이슈를 바로 잡았다.

**잘한 점**:

- "만들고 → 써보고 → 고친다" 사이클이 짧았다. v3 릴리스 후 3시간 안에 3개의 fix 커밋이 나옴.
- Reviewer 워크플로우의 commit 순서 문제는 직접 돌려보지 않으면 절대 발견 못할 버그였다. 실사용의 힘.
- Slash commands 도입으로 역할 전환 마찰이 크게 줄었다. `/review` 한 단어면 Reviewer 모드.

**교훈**:

- 워크플로우에서 **순서**는 정말 중요하다. commit이 report보다 먼저 실행되면 파일이 빠지는 건 당연한데, 문서로만 보면 놓치기 쉽다.
- 새 기능을 추가하면 기존 워크플로우에서 **참조가 연결되어 있는지** 반드시 확인할 것 (verify.md 누락 사례).

**다음에 할 것**:

- 실제 프로젝트에 이 템플릿을 적용해서 실전 피드백 수집
- auto-lint hook 실사용 테스트
- Epic 플래닝 워크플로우 실전 테스트
