# Example: Good Code Review Output

This file shows what a well-structured code review looks like.
Replace this with a real example from your project after the first successful review.

## What makes a good example
- Issues prioritized by severity (Critical > Important > Minor)
- Each issue has file:line reference and suggested fix
- Good patterns are explicitly called out (reinforces quality)
- Verdict is clear and justified
- No style nitpicks mixed with real issues
