# Example: Good Bug Fix Output

This file shows what a well-executed bug fix looks like.
Replace this with a real example from your project after the first successful bug fix.

## What makes a good example
- Root cause clearly identified (not just symptom fixed)
- Minimal change scope (only affected files touched)
- Regression test included
- Gotchas encountered during the fix are noted
- Handoff is clean and actionable
